# Omnis Desktop - v2.0.1 Early Build (Setup & Release Info)

## Release Metadata
- **Version**: 2.0.1
- **Build Status**: Early Build
- **Target OS**: Windows (nsis)
- **Release Date**: 2026-04-24

## Critical Updates & Fixes

### 1. Salestrack: Stock Pipeline & Earmarks
- **Fixed**: Potential Customers (earmarks) failed to persist or display when editing existing records.
- **Improved**: Data retrieval logic in `omnis_dashboard.py` now uses a robust join-like fetch for child table records, bypassing DocType naming conflicts ("Stock Pipeline" vs "Stock Sheet").
- **Improved**: Frontend parsing in `index.html` now handles both array and stringified JSON formats for child table data.

### 2. Versioning & UI
- **Updated**: Application version bumped to `2.0.1` in `package.json` and `app.json`.
- **Updated**: Login screen now displays `V2.0.1-EARLY BUILD` for clear identification.

## Deployment Instructions

### A. Backend (Frappe Server)
1. Upload the updated `omnis_dashboard.py` to your server.
2. Place it in: `~/frappe-bench/apps/powerstar_salestrack/powerstar_salestrack/`
3. Restart the bench to apply Python changes:
   ```bash
   bench restart
   ```

### B. Desktop Client (Build Installer)
1. Ensure all dependencies are installed:
   ```bash
   npm install
   ```
2. Build the new installer:
   ```bash
   npm run build
   ```
3. The new installer will be named `Omnis_Setup_2.0.1.exe` in the `dist/` folder.

### C. GitHub Release
1. Push the latest code to the `master`/`main` branch.
2. Create a new release tagged `v2.0.1`.
3. Upload `Omnis_Setup_2.0.1.exe` and `latest.yml` from the `dist/` folder to the release artifacts.
4. Existing users will receive the update automatically via the built-in auto-updater.
